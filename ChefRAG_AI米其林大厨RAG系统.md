## 2. 项目一句话介绍

这是一个基于 **LangGraph 状态图** 构建的 `ReAct` 架构智能膳食 Agent。它通过 7 大核心工具实现了用户画像读取、实时天气局势感知（AMap/和风等四级高可靠天气链）、食谱混合检索、做菜食材物理扣减以及口味偏好的主动记忆，并搭配了一套极具艺术设计感的 `Heritage Warmth`（传承暖意）拟物化 Web 控制面板。

---

## 3. 项目背景

在现实生活里，人们往往不知道“今天吃什么”或“冰箱里剩下的食材能做什么菜”。同时，现有的智能食谱系统大多只能根据固定关键词检索，无法考虑天气冷暖时令、用户忌口、长期饮食目标（如低碳高蛋白）以及冰箱里食材是否即将过期等多维约束。

本项目跳出了传统的“单轮 RAG 检索生成”框架，将大模型置于一个具备 **主动决策 (ReAct)** 和 **记忆持久化** 能力的 LangGraph 双节点拓扑中。项目设计了高可用性防御降级和双保险机制，可在网络抖动或向量数据库损坏时秒级降级为本地规则模式，是一套极具工程韧性的端到端 AI Agent 解决方案。

## 6. 核心功能总结

### 1. 极简 LangGraph 双节点状态图决策机
* **`agent_node`**：核心决策节点。Agent 处于 ReAct 交互循环中，分析用户意图，自发且并发地调用 7 大底层工具，感知天气局势，读取画像，查验冰箱，最终通过 RAG 召回食谱并生成回答。
* **`memory_node`**（偏好提炼与记忆压缩）：仅在会话结束（如重置或退出）时按需触发，通读多轮对话记录，通过 LLM 提炼并压缩用户的最新饮食习惯（如忌口、喜爱口味），增量更新至 `user_profile.json`。

### 2. 7 大高内聚底层 Tools 体系
* 包含 `get_user_profile`（读画像）、`update_user_profile`（写画像）、`get_fridge_inventory`（查冰箱）、`update_fridge_inventory`（添加/修改冰箱食材）、`deduct_fridge_ingredients`（物理扣除冰箱食材）、`get_weather_and_location`（实时天气与时令贴士）和 `search_recipes`（混合检索）。

### 3. 多级天气时令与高精自适应定位
* 拥有国内最精准的**四级天气自适应查询链**（高德天气 -> 心知天气 -> 和风天气自动切换 -> wttr.in 兜底），防封重试。根据当前定位的天气气温，计算今日应季推荐食材与养生小贴士。

### 4. 冰箱物理扣除与下厨条件智能识别
* **物理扣除联动**：用户在聊天界面点击“开始烹饪”或口述“我确认做冬瓜排骨汤吧”时，Agent 触发规则拦截与 `deduct_fridge_ingredients` 工具调用，直接物理扣除冰箱里的相应食材量。
* **下厨条件识别**：当用户画像处于“外卖（Delivery）”模式或在对话中透露出“在公司”、“不开火”等无下厨条件时，系统智能识别并解除“必须拥有冰箱食材”的强过滤限制，转向无门槛食谱推荐，优化特殊场景下的用户体验。

### 5. Heritage Warmth 拟物化高保真前端
* **视觉美学**：采用燕麦暖灰到日光黄的纸质渐变底色，搭配典雅衬线字族与毛玻璃卡片，呈现精品美食杂志格调。
* **拟物设计**：顶部为“实木卷轴天气挂幅”，推荐食谱卡采用“法式手写复古餐牌”，冰箱列表则被设计成按食材分类自动映射 Lucide 图标的“白色陶瓷货架抽屉”，即将过期的食材有红色呼吸闪烁警告。
* **API_BASE 智能重定向**：前端 JS 智能识别 file: 协议，无论用户是通过 `127.0.0.1:8000` 还是本地直接“双击 index.html”，皆能 100% 自动容灾定位并成功发起对话。

## 7. 项目亮点

1. **AI 决策与物理状态的深度闭环**：不仅局限于问答，大模型可以直接操作“冰箱食材增删”和“做菜扣减”，实现了真正的物理状态机变迁。
2. **极佳的工程容灾韧性（高可用降级）**：若大模型服务超时或不可用，或向量数据库 DLL 异常，系统可在 1.0 秒内无感退化至**纯本地规则检索/BM25与静态季节养生建议模式**，保证服务 100% 不掉线。
3. **多轮持久化记忆**：利用 LangGraph 状态图的 `MemorySaver` 和自定义 `memory_node`，避免频繁调用 LLM 造成高延迟，只在会话结束时“一键总结并归档偏好”。
4. **四级高可用精准天气链**：解决 wttr.in 在国内数据误差大、和风天气易 403 封锁的痛点，引入高德+心知+域名自动切换重试，展现了生产级的 API 对接水准。
5. **拟物化与高拟真流式交互**：打字机流式字符打印配有闪烁的古铜金微光游标（`▮`），冰箱卡片删除时播放 380ms 平滑向右滑出渐隐动画（`removing`），达到极高水准的设计稿视觉品质。
6. **完善的测试套件**：编写了针对 6 大场景的集成回归测试 [test_flow.py](file:///d:/C_Drive_Migrated/yyy/Desktop/chefrag/tests/test_flow.py) 与后端 API 的冒烟测试 [test_web_server.py](file:///d:/C_Drive_Migrated/yyy/Desktop/chefrag/scratch/test_web_server.py)，具备极强的工程质量控制意识。
